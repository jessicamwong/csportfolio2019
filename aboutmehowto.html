How the Project Works Overall

I created my webpage by piecing together different parts of code to create one cohesive page. I used a variety of tags to code my webpage, such as headers/headings, body tags, paragraph tags, and list tags. Tags are the specific elements of the code that define what content goes onto the webpage. This includes text, images, placement, etc.

Specifics of the Code

Headings

One of the main tags that I used for my "About Me" page was the heading tag (<h1></h1>) . This tag created all of the headings on my page such as the About Me, Introduction, Fun Facts, and Intro to Programming headings. There are six different heading tags (h1, h2, h3, h4, h5, h6) which determine the size of the heading. The h1 tag creates the largest heading while the h6 tag creates the smallest heading. For my about me page, I used the h1 tag to create the "About Me" heading because it is the main topic of my webpage. In order to create this tag, I used an opening and closing angle bracket with the h1 in between. You can think of these angle brackets as the greater than and less than symbols. I then created an end tag, which is almost the same as an opening tag, except there is a forward slash before the h1. Finally, I put "About Me" in between the opening and closing tag. I used the same process to create the rest of my headings on my page, except, I used the h2 heading tag for both my "Introduction" and "Intro to Programming" headings because they are different sections on my page. The h3 heading tag was used to create the "Fun Facts" heading because it is a sub-category of my introduction. Lists

Additionally, I used the list tag to create all of the lists that you see on the page, like the fun facts list and the list of the tags. There are two types of lists: ordered and unordered. An ordered list looks something like how the "Fun Facts" list is organized. An ordered list creates numbers for each list item. In order to create an ordered list, you have to use the ordered list tag which looks like this: <ol></ol>. The "ol" stands for ordered list. You may notice that all tags are created the same way, with an opening and closing angle bracket sandwiching the element. To begin the actual list, however, you have to create list tags and have them nested within the ordered tags. List tags look like this: <li></li>. Coding Choices and Decisions

While coding my About Me page, I had to make many choices. The majority were about the formatting and organization of my webpage. This was due to the fact that the content that appeared on my screen did not always look that way I wanted it to. For example, when I first began creating my page, I started by coding the headings and paragraphs. However the code only showed up on the webpage as it was designed to do. This meant that none of the content was centered or formatted a certain way. This was a limitation that I faced in my code because I had to separately code specific elements into my CSS. Since I wanted to center my whole page, I would normally use the "text-align: center" element. However, this function did not always work and therefore, was another limitation and stuggle that I encountered throughout the coding process. Since this function did not always work, I used margins to center my content. Here is an example of how I centered the information on webpage:

Another decision I had to make was determining the size of each heading. You can see that my About Me page has headings of different sizes, with the "About Me" heading being the biggest and the "Fun Facts" heading being the smallest. The reason behind these differences is that I wanted to correlate the heading size with importance. The "About Me" heading is the largest on my page because it is the most important, as that is the main topic of my page. The "Introduction" and "Intro to Programming" headings are still fairly large, but smaller than the "About Me" heading because they are still important, but are sections of my page that do not define the main point of my page. Lastly, the "Fun Facts" heading is the smallest because it is a subsection of my introduction, so it is not too important.
